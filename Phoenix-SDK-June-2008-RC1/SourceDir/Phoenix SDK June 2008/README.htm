Installing the Phoenix Platform SDK

Software Requirements

A PC running the following software is recommended for the Phoenix SDK.  While every effort has been made to assure a stable environment Phoenix is under development and so we strongly recommend using Phoenix on a machine dedicated to development with Phoenix.

·         Windows Vista or Windows XP Professional or Windows Server 2003

·         Visual Studio 2008, including the Express versions.

 

Installation

This is a brief summary of how to install the Microsoft® Phoenix Platform SDK onto your development machine.

 

·         You must have Administrator privileges to correctly install the Phoenix Platform SDK.

·         Do not install this Phoenix Platform SDK over previous versions of this or any other Phoenix Platform SDK.  Improper installation of the Phoenix Platform SDK can cause the use of old and incorrect files, which will lead to unreliable results. If upgrading from a previous Release, please uninstall that version before starting this process. 

·         You must have Visual Studio 2008 installed before installing the Phoenix SDK.  Be sure to uninstall any old versions of Visual Studio before installing Visual Studio 2005.

·         Run the phxRDK.msi file.

 

Verifying Your Phoenix SDK Installation

Verifying Your Phoenix SDK Installation

After installation, you will find a Microsoft Phoenix SDK June 2008 item in your Start menu.  When you run Visual Studio, you will also discover Phoenix projects under New | Projects.

Samples

 

This release adds to our growing body of samples.  Further information about them can be found in the samples readme.  We are continuously striving to add more samples that demonstrate specific features of Phoenix and how to make use of them, however we cannot be complete and so we suggest using these samples as an introduction and then reading the Phoenix Documentation for more complete information.

Several of the samples require an input executable that can be read by the Phoenix PE Reader.  The best description of what is needed can be found in the Phoenix.chm file, under Inside Phoenix | Input and Output Components | PE Reader.

 

Wizards

We have included Phoenix Wizards into the SDK in order to provide a quick-start for writing Phoenix based tools and compiler phases.  Wizards are available to build the code templates in C# or C++.  Since the Phoenix SDK only supports a managed interface, tools and phases can only be written in managed languages so the C++ wizards generate C++ which runs in the CLR environment.

 

While these wizards do not write the entire application for you, they do generate a lot of the code that would ordinarily be implemented with cut and paste from sample code.  In addition to the wizards we have included in the documentation code walk-throughs that help you understand how to use the wizard generated code to implement your phoenix based tool.

 

When building a template with the wizards the Visual Studio Project will include references to the debug versions of all referenced .dlls and will include #defines for the appropriate debug settings.  If you wish to reference the release versions of the .dlls you must manually change the reference.  By convention all debug dlls have the letter d as the last letter of the file name (before the file extension) and the release versions have the same name without the d (for example the release version of Phoenix is phx.dll,  the debug version is phxd.dll).

 

Reporting Bugs

Included in the Phoenix SDK is an item Report SDK Bugs represented by an Icon looking like an aerosol can of bug-spray.  Selecting this shortcut will start Internet Explorer and bring you to a website where you can report bugs directly into the product group bug database (this shortcut invokes a command shell and may require action on your part to allow the actions in light of mal-ware protection tools).  Before entering a bug we suggest you explore the database for existing bugs already reported to avoid duplication.

 

Once a bug has been entered it is reported to the product team which will prioritize all bugs and resolve them as appropriate.  While we cannot guarantee the prompt resolution of any bug, entering into the database notifies the team of the problem and assures the problem will be evaluated before final release of Phoenix.

Known issues

  • BBcount does not support references to strongly signed assemblies.

 

  • PEReader fails to report failure to load MSPDB80.DLL and instead reports an unhandled exception.

 

  • On 64-bit OSes, you may encounter issues when running applications that reference the Phoenix assemblies.  Because only the x86 hosted versions of the Phoenix dlls are included in the SDK, your application must also be x86.
    1. Symptoms: System.BadImageFormatException when attempting to execute an application referencing the Phoenix dlls.
    2. Workound for Visual C# 2008: Explicitly set the application to compile as x86, rather than Any CPU.
    3. Workaround for Visual C# Express: Unfortunately the Express edition does not allow switching between targets.  To get C# applications to work, you will need to run the following command from an Administrator command prompt:
      1. C:\Windows\Microsoft.NET\Framework64\v2.50727\Ldr64.exe setwow

This is a system-wide setting and will alter the CLRs behavior for applications not explicitly marked as X86 or X64.  To change back, run:

2.    C:\Windows\Microsoft.NET\Framework64\v2.50727\Ldr64.exe set64

 

·         If Visual C# Express is the only development environment installed on a machine, debug Phoenix dlls will fail to load.  This is because they reference the debug CRT which is not installed with this product:

1.    Symptom: The applications side by side configuration is incorrect.

2.    Workaround: Install Visual C++ Express 2008.

  • After installing the SDK when Visual C++ is not installed, the C++ toolset will not be available on the Phoenix command prompts. To remedy this issue, install a Visual C++ SKU, then uninstall and reinstall the SDK. A repair is not sufficient.
