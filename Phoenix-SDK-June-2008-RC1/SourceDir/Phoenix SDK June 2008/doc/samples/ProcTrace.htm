ProcTrace Sample

 

 

Sample Name: ProcTrace

 

Sample Category: Instrumentation/Code Transformation

 

Description : The ProcTrace sample consists of two parts:  the instrumentation application, and the runtime. These are located in the subdirectories ProcTraceInstr and ProcTraceRuntime respectively.

 

The sample will instrument a native (unmanaged) x86 binary by adding, to the beginning of every function, a call which passes either just the address of the function or the address and name of the function to the runtime DLL dependent upon the /funcname command-line switch.

 

Source Language: C++

 

Sample Location: Samples\ProcTraceInstr\cpp

 

Usage: From the command line:

 

ProcTraceInstr  /out <output binary>  /pdbout <output pdb>

[/funcname]  /in <input binary>

 

Detailed Description: The ProcTraceInstr sample inserts instrumentation at the entry to every method in the specified binary.  It does this by adding an import to a method in ProcTraceRuntime.dll and then inserting a call instruction with parameters into the beginning of every method. 

 

Which method is imported, and what parameters are passed out, depend on the /funcname switch.

 

The address of the instrumented method is always passed out; optionally a string containing the name of the method will be added to the binary and this will also be passed across to the runtime DLL..

 

ProcTraceRuntime.DLL is a native (unmanaged) x86 DLL that contains 3 exported methods each of which preserves:

 

  • the general purpose registers
  • the flags registers
  • the Win32 LastError value

 

across its execution so as to not alter the behavior of the instrumented application.
