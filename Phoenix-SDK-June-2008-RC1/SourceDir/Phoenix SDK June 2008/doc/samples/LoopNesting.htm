Loop Nesting

 

 

Sample Name: LoopNesting

 

Sample Category: plugin

 

Description: This plugin identifies loop headers, bodies and exits. In addition, it builds a parent-of relation between loops to represent loop nesting.

 

Source Language: C++

 

Sample Location: Samples\LoopNesting\cpp

 

Usage: Uses the standard Phoenix plugin model for C2.  Example:

 

  cd %PhoenixSdkUserRoot%\samples\LoopNesting\cpp

  devenv LoopNesting.vcproj /build

  cl -nologo -c %PhoenixSdkUserRoot%Applications\tests\src\cpp\SimpleNestedLoops.cpp -d2 plugin:debug\LoopNesting.dll

 

Detailed Description:

 

Loop headers are identified using back-edges.

 

Loop bodies are identified as the sets of nodes reachable backward from back-edge predecessors.

 

Loop exits are identified as sources of edges leading to non-loop nodes.

 

The parent-of relation between loop headers is built “inside out”: the procedure that computes loop bodies is invoked in post-order (by which nodes furthest away from the start node are processed first), so that the header of a nested loop appears as a body member which is itself a loop header.

 

Once this relation is available, the plugin also computes loop nesting levels and exclusive bodies, i.e. sets of proper loop members (exclusive of members of nested loops).
