FuncNames Sample

 

 

Sample Name: FuncNames

 

Sample Category: Plugin -- Compile-time Reporting

 

Description: The FuncNames plugIn simply reports the name of each function as it is being compiled by the c2 backend.

 

Source Language: C#, C++/CLI

 

Sample Location: Samples\funcnames\csharp,  Samples\funcnames\cpp

 

Usage: Uses the standard Phoenix plugin model for c2.  Example:

 

cl -d2plugin:<path to funcNames.dll>  ...

 

Detailed Description: The source code to compile should be native C++.  For an explanation of FuncName’s design, see the PhxDoc.chm help file, under Inside Phoenix | Core Components | Plug-ins, as well as the comments in the code.  FuncNames is designed as the simplest possible Phoenix plugIn; a template upon which to build real plugIns.

 

The reported names are the decorated names from the C++ compiler front end, since these are what Phoenix works with.  In particular, constructors, destructors, and operators have cryptic decorated names, which is why they are sometimes called “mangled names.”  If you wish, you can save the output into a file, and then pass the file to undname.exe, the “name undecorator” that ships with some versions of Visual Studio.
