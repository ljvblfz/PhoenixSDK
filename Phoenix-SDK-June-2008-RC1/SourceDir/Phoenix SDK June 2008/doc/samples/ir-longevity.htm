IR Longevity Sample

 

 

Sample Name: IR-Longevity

 

Sample Category: Plugin showing dynamic extensibility

 

Description : The IR-Longevity plugin tracks the lifetimes of instructions and operands.  At the end of compilation, a report is presented which indicates the lifetimes of the objects as a birth/death matrix of phases.

 

Source Language: C#, C++/CLI

 

Sample Location: Samples\IR-Longevity\csharp,  Samples\IR-Longevity\cpp

 

Usage: Uses the standard Phoenix plugin model for c2.  Example:

 

cl -d2plugin:<path to IR-Longevity.dll> [-d2IRLongevityOutput:<file name>] ...

 

Arguments in [] are optional:

 

IRLongevityOutput:<file name> - Dumps the output from this sample into a file provided in the command line.

  Otherwise, IR-Longevity.txt output file will be created by default.

 

Note: if you invoke c2.exe via cl and specify multiple source files on the command line, the IR-Longevity output will only show the results for one of the files.

 

Detailed Description: The IR-Longevity sample demonstrates a number of dynamic extensibility concepts. The sample is packaged as a Phoenix plugin that can be used with any phase-based Phoenix executable. Extension objects are used to store the birth phase of instructions and operands on the objects themselves; events are used to determine when an IR object is created or destroyed; a phase is added to kill all IR objects alive at the end of phase list processing.
