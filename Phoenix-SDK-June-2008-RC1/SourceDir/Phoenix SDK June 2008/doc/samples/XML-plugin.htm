XML Plugin Sample

 

 

Sample Name: XML plugin

 

Sample Category: plugin

 

Description: The XML plugin dumps each function’s IR as XML after every phase of C2.

 

Source Language: C++

 

Sample Location: Samples\XML-plug-in\cpp

 

Usage: Uses the standard Phoenix plugin model for C2.  Example:

 

cl -d2plugin:<path to XML-plug-in.dll>

   -d2xml-dir:<directory to receive XML files> ...

 

Detailed Description: This sample uses a post-phase hook to run after every executed phase of C2.  It dumps the IR as XML (but only selected properties of operands), one XML file per {phase x method}.  Each XML file’s name incorporates the phase name and the decorated method name.  An XSLT stylesheet (phx-xml.xsl) is included in the XML-plug-in sample directory for simple viewing with Internet Explorer; this file should be placed in the same directory as the XML files.  If desired, the generation of the reference to this stylesheet is easily removed in the code in order to create unadulterated XML.

 

The default directory for the xml-dir control is the current directory.  The plugin does not create the XML directory if it does not already exist.

 

A more elaborate viewer, IR Explorer, is also provided: Samples\XML-plug-in\IR-Explorer.  This is a stand-alone Windows Form application that permits browsing by phase or method, and displays the IR by basic blocks.
