The Phoenix Pascal Compiler

 

 

Sample Name: Pascal

 

Sample Category: Compiler.

 

Description: The Phoenix Pascal compiler is a sample end-to-end compiler that uses Phoenix to perform back end code generation.

 

Source Language: C++/CLI

 

Sample Location: Samples\Pascal\cpp

 

Documentation: For details about the implementation of the compiler, see The Pascal Sample Compiler in the Phoenix documentation.

 

Detailed Description: The Phoenix Pascal compiler is based on the specification that is described in Pascal User Manual and Report, Second Edition, by Kathleen Jensen and Niklaus Wirth. The compiler implements all the language features that the specification outlines, and the following language extensions:

·         An exponentiation (**) operator

·         An external directive for procedures and functions

·         An otherwise target for case statements

·         The -> operator, which can be used instead of the ^ operator

·         Multiple source file modules

 

You can compile and run the compiler by opening the solution file Pascal.sln from the Visual Studio IDE, or by using MSBuild at the command prompt.

 

After you have built the compiler, you can compile Pascal programs. The following shows the command-line options that are available to the compiler. Arguments that are enclosed in [] are optional.

   msp.exe [/c] [/clr] [/d] [/nc] [/Oe] [/p] [/v] <source-file(s)>

The following table describes the available command-line options:

 

Option

Description

/c

Compile without linking.

/clr

Enable common language runtime (CLR) support. This option is not implemented.

/d

Generate debug information.

/nc

Skip compilation.

/Oe

Enable expression optimization.

/p

Pretty-print source listing.

/v

Report AST visit tracking.

 

If you want to change the grammar files (pascal.l or pascal.y), you will need the tools Flex and Yacc. You will also have to enable the custom build steps that are associated with these files.
